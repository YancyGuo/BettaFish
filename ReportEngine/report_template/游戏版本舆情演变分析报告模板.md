### 游戏版本舆情演变分析报告模板
- **1.0 版本舆情总体概览**
    - 1.1 版本整体口碑概述（正向 / 中性 / 负向大致态势）
    - 1.2 版本舆情演变主线（从预期到实际体验的整体走向）
    - 1.3 本期关键发现与核心结论（对产品与运营的几条浓缩判断）
- **2.0 版本背景与数据边界**
    - 2.1 版本基本信息（游戏名称、版本号、版本副标题、核心更新要点）
    - 2.2 分析时间范围与阶段划分说明（阶段 A / B / C 的时间区间与含义）
    - 2.3 数据来源与样本结构概览（主要平台、内容量 / 评论量等整体规模）
- **3.0 整体声量与情绪趋势**
    - 3.1 声量走势与关键峰值（对应爆料、前瞻、上线、公告等节点）
    - 3.2 情绪分布与阶段性变化（正向 / 中性 / 负向占比的主要变化趋势）
    - 3.3 渠道差异概览（不同平台在声量与情绪上的整体差异简述）
- **4.0 分阶段舆情演变分析（A/B/C）**
    - 4.1 阶段 A：上线前（预期构建阶段）
        - 4.1.1 该阶段声量与情绪概况
        - 4.1.2 主要讨论议题与高频观点（围绕爆料 / 前瞻 / 测试服）
        - 4.1.3 重要舆情节点与代表性声音
    - 4.2 阶段 B：上线首周（体验落地阶段）
        - 4.2.1 首周声量与情绪变化概况
        - 4.2.2 玩家实际体验相关的核心议题（角色、玩法、活动、性能等）
        - 4.2.3 关键事件与舆论集中反馈
    - 4.3 阶段 C：版本中后期（口碑沉淀阶段）
        - 4.3.1 中后期声量与情绪整体走向
        - 4.3.2 长尾议题与持续争议点
        - 4.3.3 阶段性总结（对本版本“最终印象”的形成过程）
- **5.0 核心议题与体验维度综述**
    - 5.1 角色与数值相关议题概览（强度评价、平衡争议等）
    - 5.2 剧情与世界观相关议题概览（故事节奏、文本质量、演出反馈等）
    - 5.3 活动与玩法相关议题概览（可玩性、肝度、奖励感受等）
    - 5.4 商业化与氪金体验相关议题概览（卡池、礼包、付费体验等）
    - 5.5 技术与服务体验相关议题概览（性能、BUG、服务器、客服等）
- **6.0 风险提示与机会点洞察**
    - 6.1 当前版本暴露出的主要风险点（容易在后续版本或社区发酵的问题）
    - 6.2 可以放大的正向机会点（玩家明显认可的设计或改动方向）
    - 6.3 面向后续版本与运营的行动建议要点（从舆情视角给出的优先事项）
